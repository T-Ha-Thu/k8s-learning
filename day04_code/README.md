- Write the challenges of using standalone containers.  

Standalone containers are good for simple, single-app deployments, but they become difficult to manage, scale, and maintain in production environments.  
  
- Describe how Kubernetes solves those challenges  
  
Kubernetes turns containers into resilient, scalable, and manageable units. What was manual and error-prone with standalone containers becomes automated, monitored, and easier to maintain.  
  
- Use Kubernetes when you need scalability, reliability, and automation.  
  
Avoid Kubernetes when simplicity, speed, and minimal resources are more important.
